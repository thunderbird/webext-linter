Privileged core symbol in WebExtension code error

core-symbol-in-webext

What it detects

WebExtension code — background, content, options and popup scripts — runs in a sandbox with no chrome privileges. Privileged Thunderbird internals such as Services, ChromeUtils and the Components / Cc / Ci / Cu family simply do not exist there.

The check looks only at the developer's own files that are actually reachable as plain WebExtension code. If one of those files refers to a privileged global, that reference is either a bug or privileged code that has been mis-packaged as a WebExtension file. A name that the file defines or imports itself is the developer's own symbol, so it is left alone.

When it fires The add-on is told it uses privileged Thunderbird internals in a WebExtension context where they are not available, and must remove the usage.

Source of truth: src/checks/rules/core-symbol-in-webext.js and its entry in assets/registry.yaml.
