Modified or outdated Experiment draft error

experiment-modified

What it detects

A bundled Experiment that is a recognised published Thunderbird API draft, but is not the unmodified latest version — it has been changed locally or is an older draft. The submission keeps full review feedback but is rejected until the unmodified latest upstream copy is bundled.

It reads the per-experiment classification computed earlier in the pipeline and raises one error per experiment marked as modified. It stays silent for non-Experiments, for pristine experiments, and for unsupported ones — those last are handled in the invalid-experiment phase by the experiment-not-allowed check.

When it fires The developer is told the named API is a published Thunderbird Experiment draft but this add-on bundles a modified or outdated copy, and is asked to bundle the unmodified latest version from the upstream repository.

Source of truth: src/checks/rules/experiment-modified.js and its entry in assets/registry.yaml.
