Minimize host permissions info

minimize-host-permissions

What it detects

This check looks for very broad host permissions that the manifest requests as required, meaning they are granted at install time. A broad pattern is the all-URLs pattern or any pattern whose host part is a bare wildcard, which grants access to every site under that scheme.

It scans both the MV3 host-permissions list and the host match patterns that appear in the MV2 permissions array. Named permissions in that array are not hosts and are ignored here, as are well-formedness questions about a pattern, which other checks handle. Each distinct broad pattern is reported once, anchored to its line in the manifest.

When it fires The add-on is told it requests broad host permissions that are granted at install, and is advised to request only the specific hosts it needs — or, if broad access is genuinely required, to declare it as an optional host permission the user grants on demand.
