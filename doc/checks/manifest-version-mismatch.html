Manifest version mismatch error

manifest-version-mismatch

What it detects

The add-on's declared manifest_version disagrees with the schema set it is being reviewed against — for example an MV2 add-on reviewed against the MV3 schema, or the reverse. When the two do not line up, the submission is invalid.

The check needs a manifest that actually parsed and a numeric manifest_version, and it needs to know which major version the active schema set targets. It simply compares those two numbers and reports only when they differ.

When it fires The add-on is told its manifest.json is invalid because the declared manifest version does not match the schema set being applied, with a link to the WebExtension API documentation.

Source of truth: src/checks/rules/manifest-version-mismatch.js and its entry in assets/registry.yaml.
