Missing third-party library reference error

missing-library

What it detects

This check flags a bundled JavaScript or CSS file that looks like a distributed third-party library but is not declared in a VENDOR file. It recognises libraries by surface signals: a minifier banner comment, a UMD/AMD module wrapper, a .min.js or .min.css name, or a well-known library filename such as jQuery or Bootstrap.

The detection is heuristic and hash-free, so it cannot say which library or which version a file is — only that the file appears to be a library that should be linked to its upstream source. Minified or obfuscated code that carries no library signals is a different situation, reported by the obfuscated-code check, so this check leaves those to it.

When it fires The add-on is told the named files appear to be third-party libraries and is asked to link each to its source and ensure the bundled copy matches the linked version, commonly via a VENDOR file at the add-on root.

Source of truth: src/checks/rules/missing-library.js and its entry in assets/registry.yaml.
