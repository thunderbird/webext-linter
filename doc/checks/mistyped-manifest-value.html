Mistyped manifest value warning

mistyped-manifest-value

What it detects

A manifest key the schema does know about, but whose value has the wrong type or shape — for instance a number where a string is expected, a value outside an allowed range, or a list of the wrong length. Thunderbird misreads or ignores such entries, so this is a warning.

The manifest is validated against a JSON Schema derived from the annotated manifest schema. The check is deliberately conservative: it only reports concrete, low-noise problems — wrong type, disallowed enumerated value, out-of-range number, and wrong list length — and ignores broader structural complaints. It never throws, and if the schema cannot be built or compiled the check quietly disables itself so a translation gap can never wrongly flag a valid manifest.

When it fires The add-on is told that a named manifest entry has a problem with its value and that Thunderbird ignores or misreads such entries, with a link to the WebExtension API documentation.

Source of truth: src/checks/rules/mistyped-manifest-value.js and its entry in assets/registry.yaml.
