Check for a missing payment disclosure manual

missing-payment-disclosure

What it detects

This is a human-review item with no automated logic behind it. A reviewer determines whether the add-on requires a subscription or other form of payment in order to function.

If a payment is in fact required, the reviewer then checks whether the "needs payment" flag on the add-on's ATN listing page is enabled. A payment requirement that is not disclosed through that flag is the problem this item catches.

Decision flow

flowchart TD
  A["Reviewer tests the add-on and
checks its ATN listing"] --> B{"Does it require a subscription
or other payment to function?"}
  B -- no --> OK([no finding])
  B -- yes --> C{"Is the 'needs payment' flag
on the ATN listing enabled?"}
  C -- yes --> OK
  C -- no --> M([flag for the developer / manual finding])

  classDef ok fill:#e9f5ee,stroke:#2c7a7b,color:#1d4d4d;
  classDef manual fill:#eceef1,stroke:#5b626b,color:#3a3f45;
  class OK ok;
  class M manual;
When it fires The developer is told a payment appears to be required to use the add-on or its service, and is asked to update the payment status to yes.

Source of truth: the manual-checks entry in assets/registry.yaml.
