Unknown permission error

manifest-unknown-permission

What it detects

A value listed under permissions or optional_permissions that the platform does not recognise. A valid entry is either a known permission, a known data-collection permission, or a host match pattern. Anything else makes the manifest invalid.

The check runs only on a parsed manifest. It looks at every string in the two permission lists and checks it against the schema's known permission sets and the match-pattern rules. Non-string entries are ignored, and each unknown value is reported individually, anchored to its line in manifest.json where possible.

When it fires The add-on is told its manifest.json is invalid because one or more requested permissions are unknown, with a link to the WebExtension API documentation.

Source of truth: src/checks/rules/manifest-unknown-permission.js and its entry in assets/registry.yaml.
