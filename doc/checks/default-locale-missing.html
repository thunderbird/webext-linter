Missing default_locale error

default-locale-missing

What it detects

An add-on that ships a packaged _locales directory but does not declare a default_locale key in its manifest. The default locale names the fallback language for the localized strings, and without it Thunderbird refuses to load the add-on.

This is the mirror image of the unused-default_locale check, which covers the opposite mistake of declaring a default locale with no _locales directory. This check only acts when a _locales directory is actually present; if the manifest did not parse, or there is no _locales directory, or a default locale is already declared, it stays silent.

When it fires The add-on is told a _locales directory is present but the manifest declares no default_locale, so Thunderbird will refuse to load it, and is asked to add a default_locale naming the fallback locale such as "en".

Source of truth: src/checks/rules/default-locale-missing.js and its entry in assets/registry.yaml.
