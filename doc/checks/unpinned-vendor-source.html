Unpinned vendored source error

unpinned-vendor-source

What it detects

A VENDOR-declared file whose source is on a trusted host but is not pinned to an immutable reference — for example a link to a branch like main, or an unversioned URL. Such a source can change after review, so the bundled bytes cannot be verified against it.

Before the review, each declared source URL is classified offline. A source on a trusted host counts as pinned only when it names a specific version, tag, or full commit hash; a branch or an unversioned link does not. Sources that are pinned move on to the byte comparison, and sources on untrusted hosts are routed elsewhere. The trusted-but-not-pinned sources are recorded, and this check turns each one into a finding.

When it fires The add-on is told the vendored file cites a source that is not pinned to an immutable version — a branch or unversioned URL — and is asked to link to a specific version, tag, or commit so the file can be verified against the exact bytes it was copied from.
