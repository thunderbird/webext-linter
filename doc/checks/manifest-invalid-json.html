Invalid manifest JSON error

manifest-invalid-json

What it detects

A manifest.json is present at the add-on root but is not valid JSON, so the submission cannot be read. This is the single finding in that case — every other manifest check needs a parsed manifest, so they stay silent.

The actual parsing happens when the add-on is loaded; this check simply reports the recorded parse error.

Decision flow

flowchart TD
  A["manifest.json at the add-on root"] --> B{"Did it fail to parse as JSON?"}

When it fires The add-on is told manifest.json is not valid JSON, with a link to the WebExtension API documentation.

Source of truth: src/checks/rules/manifest-invalid-json.js (parse error recorded by src/addon/load.js); wording in assets/registry.yaml.
