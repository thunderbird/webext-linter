Missing English localization warning

missing-english-localization

What it detects

English is Thunderbird's default language, so an add-on whose user-facing text is not available in English leaves most users unable to read it. This deterministic check decides, from the add-on's localization directories or from its hardcoded text, whether an English version is present.

If the add-on uses a localization directory, the answer is settled by whether an English directory exists. If it ships no localization directories, the check detects the language of the hardcoded user-facing text — the manifest name and description plus the visible text of every packaged HTML page — with a language-detection library. Confident English passes and confident non-English is a finding. When there is too little text to trust, or English nearly ties the top language, the call is too close to make automatically and is escalated to manual review.

When it fires The add-on is asked to include an English localization, since English is Thunderbird's default language and most users who do not understand the hardcoded language can read English, with a pointer to the i18n API and an example. Low-confidence cases are instead handed to a human reviewer (and, when the full summary runs, re-judged by the recheck consumer below).

Source of truth: src/checks/rules/missing-english-localization.js and its entry in assets/registry.yaml.
