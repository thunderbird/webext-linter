Acceptable Use Policy manual

acceptable-use-policy

What it detects

This is a human-review item with no automated logic behind it. A reviewer reads the listing's name, summary, description and screenshots.

Each of these is weighed against Mozilla's Acceptable Use Policy. The item catches listings whose visible content breaches that policy.

Decision flow

flowchart TD
  A["Reviewer checks the name, summary,
description, and screenshots"] --> B{"Does any of it violate Mozilla's
Acceptable Use Policy?"}
  B -- no --> OK([no finding])
  B -- yes --> M([flag for the developer / manual finding])

  classDef ok fill:#e9f5ee,stroke:#2c7a7b,color:#1d4d4d;
  classDef manual fill:#eceef1,stroke:#5b626b,color:#3a3f45;
  class OK ok;
  class M manual;
When it fires The developer is told that the name, summary, description, and/or screenshots contain content that does not comply with Mozilla's Acceptable Use Policy.

Source of truth: the manual-checks entry in assets/registry.yaml.
