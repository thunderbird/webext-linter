Remote data transmission needs a privacy policy manual

privacy-policy

What it detects

An add-on that transmits data to a hardcoded remote server. When this happens, the listing must carry a privacy policy that discloses what is collected — but that policy lives in the listing field on the add-on site, not in the package, so it cannot be verified automatically. The trigger is deterministic, but the outcome is escalated to a human reviewer rather than reported as a finding.

The check looks only at the developer's own JavaScript and only at overt transmissions to a literal remote host. A purely dynamic destination is left to the data-exfiltration review (which asks about consent), and a covert disguised channel is already a hard error in the disguised-* checks. This is the disclosure angle; it collects every distinct remote host and escalates them for manual confirmation.

When it fires A reviewer is asked to confirm the listing carries a privacy policy that details which user data is sent and to which services. The developer guidance explains that the policy must be the actual text pasted into the add-on site field — not a link to a remotely hosted policy — and that it must describe how the extension collects, uses, stores, shares or discloses information about people.
