Minimize web_accessible_resources warning

minimize-web-accessible-resources

What it detects

Resources exposed through web_accessible_resources that widen the add-on's attack surface more than necessary. Two concerns: an over-broad exposure pattern (for example a wildcard exposing the whole add-on, or an MV3 match like <all_urls> exposing it to every site), and a concretely-exposed file that no web-facing context in the add-on actually loads.

A deterministic pre-flight resolves the clear cases: over-broad patterns and matches become findings outright, and a concrete exposed file that is web-reachable is cleared. An exposed file that nothing web-facing loads, and that is named only by dead code with no dynamic loader, is reported as needless. When live code names the file or the add-on builds load paths at runtime, each suspected loader site is handed to the model, which judges whether that site exposes the file to a web context; the exposure is needless only if no site does.

When it fires The add-on is told one or more web_accessible_resources entries may not need to be exposed, that files belong there only when a web page or content script must reach them, and is asked to remove the unnecessary entries to reduce its exposed surface.

Source of truth: src/checks/rules/minimize-web-accessible-resources.js and its entry in assets/registry.yaml.
