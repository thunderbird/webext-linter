Check for "No Surprises" policy violations manual

no-surprises-policy

What it detects

This is a human-review item with no automated logic behind it. A reviewer runs the add-on in a test profile and reads through its code, looking for features or behavior that the user would not expect.

The concern is anything that is not documented on the ATN listing page and could surprise the user. Comparing the submission against the last accepted version with a diff can make any newly added, undisclosed behavior easier to spot.

When it fires The developer is told the add-on includes a feature unrelated to its primary function and not mentioned in the ATN description, and that such a feature must be strictly opt-in.

Source of truth: the manual-checks entry in assets/registry.yaml.
