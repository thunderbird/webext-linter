CSP allows 'unsafe-eval' error

csp-unsafe-eval

What it detects

The manifest's content security policy allows the 'unsafe-eval' keyword. That keyword re-enables eval and the other string-to-code paths the policy is meant to block, permitting dynamic code execution, which is not allowed.

The shared dynamic-execution scan parses the manifest's content security policy and records whether it allows 'unsafe-eval'. This check reports a single finding when that flag is set, anchored to the policy line in manifest.json.
