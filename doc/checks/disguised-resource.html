Disguised data transmission (resource) error

disguised-resource

What it detects

Data smuggled out of the add-on by hiding it inside a resource-load URL. Setting an element's image, iframe or media source — directly or through a setAttribute on a src or href attribute — to a remote address that is built up with runtime data appended to it sends that data to the remote host while looking like an ordinary resource load.

A resource URL is not a channel meant for transmitting data. Disguising a data send this way is treated as an error regardless of any user consent. The check looks only at the developer's own JavaScript and fires only when the URL is covert, points somewhere remote, and actually carries data — either a dynamic value concatenated into a remote URL or a user-data API call sitting in the argument. A plain static remote resource load with no data attached is not a finding.

When it fires The add-on is told it appears to send data through a resource URL such as an image, iframe or media source — a channel not meant for transmission — and that disguising a send this way is not allowed even with consent. It is asked to use a normal request and disclose what is sent.
