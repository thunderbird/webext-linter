vendor-unverified

What it detects

Third-party declarations that could not be settled automatically and need a human to verify them by hand. This has no automated severity — each case is escalated to manual review rather than reported as a finding.

Before the review, each VENDOR declaration is classified and, where possible, fetched and compared. Four situations cannot be settled automatically: a declaration with no source URL, a source on a host the linter may not fetch from, a source that could not be fetched, and a library that could not be confirmed as widely used. A VENDOR file that is present but could not be parsed at all is escalated too. This check reads those outcomes and raises one manual-review item per case.

When it fires The reviewer is asked to confirm by hand that each listed file is an unmodified copy of a widely-used library obtained from a trustworthy source. The developer is advised to give the library's exact upstream version and a source URL that pins that release on a trusted host (unpkg, jsDelivr, an npm-registry tarball, or a raw GitHub URL) so it can be verified automatically.
