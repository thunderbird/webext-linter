Modified vendored library error

vendor-modified

What it detects

A declared third-party file whose bytes do not match what its pinned source actually serves: it appears to have been modified from the upstream release.

Before the review, every trusted and pinned VENDOR source is fetched once and compared against the packaged file. The comparison tolerates end-of-line differences. A single-file source is byte-compared against the fetched copy; a whole-package tarball source is matched by hashing each file inside it. A file that matches is recorded as verified; one that does not match is recorded as modified. This check reads those results, narrates the verified files as passes, and turns each modified result into a finding.

When it fires The add-on is told a bundled third-party file does not match the bytes its declared source serves and appears to have been modified, and is asked to ship the file exactly as the source publishes it or to correct the declared source to the release the file actually came from.

Source of truth: src/checks/rules/vendor-modified.js and its entry in assets/registry.yaml.
