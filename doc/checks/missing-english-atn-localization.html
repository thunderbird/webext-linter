Missing English ATN localization manual

missing-english-atn-localization

What it detects

This is a human-review item with no automated logic behind it. A reviewer checks which languages the add-on's ATN listing page is available in.

English is Thunderbird's default language, so the listing is expected to have an English version even when its primary localization is in another language. The item catches listings that are missing that English version.

When it fires The developer is told the ATN listing should also have an English localization, since English is Thunderbird's default language.

Source of truth: the manual-checks entry in assets/registry.yaml.
