Remote eval error

remote-eval

What it detects

Code whose source may be remote being executed dynamically — the statically-undecidable pattern of fetching something and then running it, such as feeding the result of a fetch into eval. Executing remotely fetched code is not allowed, because that code never goes through review.

The definite dynamic-execution cases — a literal eval, the Function constructor, a string-valued timer, and unsafe-eval or unsafe-inline in the content security policy — are handled by separate deterministic checks. This check covers only the ambiguous case: a deterministic scan finds each fetch-then-execute site, and because whether the executed code is remote cannot be decided statically, each site is handed to the model, which judges whether the code is fetched from a remote source.

When it fires The add-on is told that remotely fetched code is executed at the named location, that remote code execution is not allowed, and is asked to bundle all code with the add-on so it can be reviewed.
