CSP allows 'unsafe-inline' error

csp-unsafe-inline

What it detects

The manifest's content security policy allows the 'unsafe-inline' keyword. That keyword permits inline scripts to run, which is a form of dynamic code execution, and is not allowed.

The shared dynamic-execution scan parses the manifest's content security policy and records whether it allows 'unsafe-inline'. This check reports a single finding when that flag is set, anchored to the policy line in manifest.json.

When it fires The add-on is told that dynamic code execution via a content security policy that allows 'unsafe-inline' is not allowed.
