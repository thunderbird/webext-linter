Missing ATN description or details manual

missing-atn-description

What it detects

This is a human-review item with no automated logic behind it. A reviewer reads the add-on's ATN listing page to judge how well it explains the add-on to users.

A good listing includes detailed usage instructions, points out where the add-on's entry points are, and provides screenshots. The item catches listings that leave these out.

Decision flow

When it fires The developer is told the ATN listing should include detailed usage instructions, entry points, and screenshots.

Source of truth: the manual-checks entry in assets/registry.yaml.
