Missing required manifest key error

manifest-missing-key

What it detects

A required top-level manifest key is absent. Every add-on must declare manifest_version, name and version; if any of these is missing the submission is invalid.

The check runs only on a manifest that parsed successfully. Each missing required key is reported separately. Unknown permissions and a manifest-version mismatch are handled by their own dedicated checks.

Decision flow

flowchart TD
  A["manifest.json"] --> B{"Did it parse successfully?"}

When it fires The add-on is told one or more required manifest properties are missing, with a link to the WebExtension API documentation.

Source of truth: src/checks/rules/manifest-missing-key.js and its entry in assets/registry.yaml.
