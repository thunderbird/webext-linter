Remote resources error

remote-resources

What it detects

Code loaded from somewhere other than the add-on's own package. Remote scripts and libraries cannot be reviewed, so everything the add-on runs must be bundled with it.

A deterministic pre-flight scans HTML, CSS, JavaScript and the manifest for remote loads: remote <script>/<link>/<iframe> and media in HTML, remote @import/url() in CSS, remote import()/importScripts() and module imports, runtime <script> injection, remote WASM, and a content security policy that permits a remote script source. Each of those definite remote loads is a finding directly. Cases static analysis cannot resolve — a non-literal URL, or an inline data:/blob: script source — carry the offending file to the model, which judges whether the loaded code is remote.

When it fires The add-on is told that remote code is loaded from the named source, that remote scripts and libraries are not allowed, and is asked to bundle all code with the add-on so it can be reviewed.

Source of truth: src/checks/rules/remote-resources.js and its entry in assets/registry.yaml.
