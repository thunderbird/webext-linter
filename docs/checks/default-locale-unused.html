Unused default_locale error

default-locale-unused

What it detects

An add-on that declares a default_locale key in its manifest but ships no _locales directory to back it up. A default locale points at localized message files that must exist in the package, and without them Thunderbird refuses to load the add-on. The finding is located on the default_locale line in the manifest.

This is the mirror image of the missing-default_locale check, which covers the opposite mistake of shipping a _locales directory with no default locale declared. This check only acts when a default locale is declared; if the manifest did not parse, or no default locale is declared, or a _locales directory is already present, it stays silent.

When it fires The add-on is told the manifest declares default_locale but ships no _locales directory, so Thunderbird will refuse to load it, and is asked to add the _locales message files or remove default_locale.

Source of truth: src/checks/rules/default-locale-unused.js and its entry in assets/registry.yaml.
