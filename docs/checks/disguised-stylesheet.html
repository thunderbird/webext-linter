Disguised data transmission (stylesheet) error

disguised-stylesheet

What it detects

Data smuggled out of the add-on by hiding it inside a stylesheet or a CSS url() reference. Setting a style property such as a background image or raw CSS text to a remote address that is built up with runtime data sends that data to the remote host while looking like an ordinary style being applied.

A stylesheet URL is not a channel meant for transmitting data. Disguising a data send this way is treated as an error regardless of any user consent. The check looks only at the developer's own JavaScript and fires on the strong case: a covert style URL to a remote host with a user-data API call (message content, contacts, credentials and the like) sitting in it. The weaker case — merely a runtime value appended to a remote URL, common in legitimate code — is left to the disguised-transmission LLM check. A plain static remote background image with no data attached is not a finding.

When it fires The add-on is told it appears to send data through a stylesheet or CSS url() — a channel not meant for transmission — and that disguising a send this way is not allowed even with consent. It is asked to use a normal request and disclose what is sent.
