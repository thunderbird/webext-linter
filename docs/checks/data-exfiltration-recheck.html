User-data exfiltration (recheck) error

data-exfiltration-recheck

What it detects

This is the post-summary recheck consumer for the data-exfiltration check. It re-judges each remote-transmission site the narrow per-site view could not clear — the residual ambiguous items — now with the whole add-on in view rather than just the transmitting file and the options page.

It runs only in the post-summary phase. When a add-on summary is produced, those unresolved sites are appended to it under this entry's summary rubric and re-judged, so a user opt-in defined in any settings page, background flag or stored preference becomes visible — not only the options page in the original narrow corpus. As before, transmitting user data to a remote host is allowed only with an active opt-in, and Thunderbird has no built-in consent prompt. With no summary, nothing is handed over, this check emits nothing, and the producer's manual items stand.

When it fires The add-on is told the listed code locations send user data to a remote server without an explicit opt-in, and that user data may be transmitted only when the user actively enables it on an options page that discloses what is sent.
