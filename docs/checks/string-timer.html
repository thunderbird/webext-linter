Code-string timer error

string-timer

What it detects

A code string handed to setTimeout or setInterval in the add-on's own JavaScript. When the first argument is a string rather than a function, the timer evaluates it as code, so it is dynamic code execution and is not allowed.

A single shared scan walks authored JavaScript for the dynamic-execution signals and tags each hit by kind. It looks only outside the pure WebExtension tree: a WebExtension sandbox cannot evaluate a code-string timer unless the manifest CSP allows it — and that permissive CSP is reported separately by the csp-unsafe-eval check — so only privileged Experiment / non-WebExtension code is scanned here. Bundled third-party libraries are excluded too. This check reports each hit tagged as a code-string timer, one finding per site.

When it fires The add-on is told that dynamic code execution via a code string passed to setTimeout or setInterval is not allowed.
