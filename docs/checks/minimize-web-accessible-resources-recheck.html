Minimize web_accessible_resources (recheck) warning

minimize-web-accessible-resources-recheck

What it detects

This is the post-summary recheck consumer for the minimize-web-accessible-resources check. It re-judges the exposed entries that the per-site analysis could not clear — the residual ambiguous items — now with the whole add-on in view rather than one referencing site at a time.

It runs only in the post-summary phase. When a add-on summary is produced, those unresolved entries are appended to it under this entry's summary rubric and re-judged: web_accessible_resources is needed only when something outside the add-on — a content script, a real web page, or another extension — reads the file; the add-on opening its own file or page from its own contexts never needs it. With no summary, nothing is handed over, this check emits nothing, and the producer's manual items stand.

When it fires The add-on is told one or more web_accessible_resources entries may not need to be exposed, that files belong there only when a web page or content script must reach them, and is asked to remove the unnecessary entries to reduce its exposed surface.
