Disguised data transmission (recheck) error

disguised-transmission-recheck

What it detects

This is the post-summary recheck consumer for the disguised-transmission check. It re-judges each covert sink the narrow per-location view could not clear — the residual ambiguous items — now with the whole add-on in view rather than just the one suspected site.

It runs only in the post-summary phase. When a add-on summary is produced, those unresolved locations are appended to it under this entry's summary rubric and re-judged, so context elsewhere in the add-on can show whether a dynamic remote URL is a legitimate load or a covert data send. Each verdict maps directly: a legitimate dynamic URL is dropped, a smuggled send becomes an error, and anything still unresolved goes to manual review. With no summary, nothing is handed over, this check emits nothing, and the producer's manual items stand.

When it fires The add-on is told the listed code locations appear to send data through a resource or navigation URL — a channel not meant for transmission — that disguising a send this way is not allowed even with consent, and that it should use a normal request and disclose what is sent.
