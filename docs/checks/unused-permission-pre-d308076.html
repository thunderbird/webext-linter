Unused permission (pre-D308076) warning

unused-permission-pre-d308076

What it detects

This is the post-summary recheck consumer for the pre-D308076 path (minimum Thunderbird version below 154, or absent). It does not enumerate anything itself: the unused-permission-manual-pre-d308076 producer hands it every declared permission a reachable API call could not be shown to require, and the whole-add-on summary pass re-judges each one with full context.

For each handed-over permission it maps the summary's verdict: a justified verdict drops the permission, an unused verdict becomes a warning that the permission appears unused, and an unsure verdict — or no verdict at all, when the summary was skipped or failed — routes the permission back to manual review. The mechanics are identical to the post-D308076 consumer; only the summary rubric differs, keeping the tabs permission justified whenever the code filters a tab query by url or title anywhere, because before the fix that needs tabs even for the add-on's own pages. With no summary nothing is handed over, so this check emits nothing and the producer's reminder stands instead.

When it fires The add-on is told that a specific permission appears to be unused, with the summary's reason, and is asked to remove it or to say which part of the add-on stops working without it.
