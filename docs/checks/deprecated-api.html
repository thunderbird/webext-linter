Deprecated WebExtension API info

deprecated-api

What it detects

Calls to WebExtension APIs that still exist in the targeted Thunderbird version but are marked deprecated in the schema. These keep working for now, but should be migrated to a supported API.

Each deprecated function, event or property is reported once. When the schema carries a deprecation message, that migration note is surfaced with the finding. Only reachable WebExtension code is examined; APIs that are absent or unsupported are handled by the unknown-api check instead.

When it fires The add-on is told the named API is deprecated in the targeted Thunderbird version and should be migrated to a supported API.

Source of truth: src/checks/rules/deprecated-api.js and its entry in assets/registry.yaml.
