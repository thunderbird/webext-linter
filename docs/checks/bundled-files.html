Missing referenced file error

bundled-files

What it detects

A file the add-on points to but does not actually ship. References are gathered from two places: manifest entries (content script JS and CSS, background scripts, pages and service workers, options pages, and action popups) and file-loading API calls in the code (icon, theme and menu registration, script and CSS injection, packaged-URL lookups, tab creation, popup assignment, and so on).

Only references that look like packaged files are judged — relative or root-relative paths with no URI scheme. Remote URLs, pseudo-schemes such as moz-extension or about, and bare fragments are left to other checks. Manifest references are anchored to the line that cites them; page- relative loader references are resolved against the calling script's host page directory, the rest against the package root. Each candidate that is absent from the package produces a finding.

When it fires The add-on is told that the named path is referenced but not bundled, and asked to include every referenced file in the package.

Source of truth: src/checks/rules/bundled-files.js and its entry in assets/registry.yaml.
