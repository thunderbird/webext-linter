Build registry override error

build-registry-redirect

What it detects

A source-code submission whose build configuration points the package install at a registry other than the default public one. A legitimate build never needs to set a registry, because the package manager already defaults to the public registry, so any registry setting is rejected outright.

This check applies only to source-code submissions, where the reviewer builds the add-on from the readable source. It scans every build-config dotfile that names a registry, at the root or in a subfolder, because a build may run from a nested directory. A custom registry can pull the same declared package names from an undeclared source, leaving the declared dependencies and the lock file a clean decoy. The value is not inspected — the mere presence of a registry setting is the rejection.

When it fires The add-on is told that its build configuration sets a package registry, which is not allowed, and that the setting must be removed so the build resolves every dependency from the default public npm registry. It is warned that a custom registry pulls the declared package names from an undeclared source, making the declared dependencies and the lock file a decoy.

Source of truth: src/checks/rules/build-registry-redirect.js and its entry in assets/registry.yaml.
