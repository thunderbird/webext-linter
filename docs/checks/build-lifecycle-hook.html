Build install hook warning

build-lifecycle-hook

What it detects

A source-code submission whose package.json declares an npm install lifecycle hook — a preinstall, install, postinstall, prepare, prepublish, prepublishOnly, or prepack script. These run automatically when the reviewer installs the declared dependencies, before the build even starts, so a hook that fetches and runs remote code is a supply-chain vector the dependency audit — which only reads the declared package.json and lock file — never sees.

This is a deterministic, offline backstop for the same concern the build process review asks a model about: it needs no token and always runs. It is a warning, not a hard reject, because legitimate hooks exist (a husky prepare, a local codegen step) — it points the reviewer at each hook to confirm it only runs local build steps and does not reach out to the network. Each declared install hook becomes one warning finding, anchored at its line in package.json.

When it fires The add-on is told that the package.json declares an install lifecycle hook that runs when the reviewer installs the dependencies, before the build, and is asked to confirm it only runs local build steps and does not fetch or execute remote code (curl/wget, a download-and-run, or a git clone of an unpinned repo).

Source of truth: src/checks/rules/build-lifecycle-hook.js and its entry in assets/registry.yaml.
