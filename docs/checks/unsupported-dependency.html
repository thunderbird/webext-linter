Unsupported dependency source error

unsupported-dependency

What it detects

A dependency declared in package.json from a source the review cannot identify or verify. Only two sources can be vetted: a pinned npm package (an exact version, or a range a committed lock file pins) and a GitHub URL, which is gated by popularity. Everything else is rejected.

Before the review begins, the vendor resolver reads the package.json dependencies once and classifies each by its source. A local file reference, a link or workspace reference, an npm alias, a tarball URL, or a git source that is not GitHub cannot be resolved to a known upstream, so it is set aside as unsupported. This check reads that set and turns each entry into a finding, anchored at its declaration line.

When it fires The add-on is told a bundled dependency is declared from a source the review cannot identify or verify, that only pinned npm packages and GitHub sources are supported, and that it should re-declare the dependency as a pinned npm package or a GitHub source, or bundle the library with the add-on as authored code so it can be reviewed directly.

Source of truth: src/checks/rules/unsupported-dependency.js and its entry in assets/registry.yaml.
