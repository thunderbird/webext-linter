Modifying innerHTML; use Element.setHTML() info

unsafe-html

What it detects

Any write to an HTML sink — innerHTML, outerHTML, srcdoc or insertAdjacentHTML. The only sanctioned way to insert markup is the Sanitizer API (Element.setHTML()), so a sink write is flagged regardless of where the content comes from or whether it was run through a sanitizer — a sanitizer is not a pass.

This is advisory (info) for now: writing markup to a sink will no longer be permitted after the release of ESR 153, and migrating to setHTML() needs Thunderbird 148+, so the check points the reviewer at each site ahead of the deadline. Every sink write is flagged; the sole exception is an empty-string or null clear (el.innerHTML = ""), which writes no content. Bundled, minified or otherwise non-authored files are skipped. Each remaining sink hit becomes a finding naming the sink involved.

When it fires The add-on is told that HTML is written via the named sink and that the only sanctioned way to insert markup is the Sanitizer API (Element.setHTML(), Thunderbird 148+) — sanitizing the content is not a pass — with textContent for plain text or a template/CSS for existing markup, and a link to the guide.
