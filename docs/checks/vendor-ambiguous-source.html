Ambiguous vendored source error

vendor-ambiguous-source

What it detects

A single source URL in the VENDOR file that is declared as the origin of more than one bundled file. A file source can only verify one file against the bytes at that URL, so pairing it with several files is ambiguous — there is no way to know which file the source is meant to attest.

The fix is to give each bundled file its own distinct source URL, or — if the files genuinely come from one place — to declare the containing directory once as a single directory source. The vendor resolver pulls these ambiguous pairings out of the VENDOR file (it does not attempt to verify them) and this check turns each into an error.

When it fires The add-on is told that one source URL is declared for several bundled files, that a file source can verify only one file, and that it must either give each file its own source URL or declare the containing directory with a single directory source.
