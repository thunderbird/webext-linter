Forked add-on manual

forked-add-on

What it detects

This is a human-review item with no automated logic behind it. A reviewer checks whether the submission is a fork of an existing add-on that is still being actively maintained.

Forks are only allowed on ATN when their name clearly distinguishes them from the original and they offer a significant difference in functionality or code. Exceptions apply when the original is no longer maintained. This item is skipped when reviewing an update against a --diff-to baseline, since an established listing is not a new fork.

Decision flow

flowchart TD

  C -- yes --> OK

When it fires The developer is told the submission appears to be a fork of an actively maintained add-on that is not clearly distinguished with significant differences, and is encouraged to contribute improvements to the original project instead.

Source of truth: the manual-checks entry in assets/registry.yaml.
