Source submission not required error

build-source-redundant

What it detects

A source-code submission whose build does no real building: its only work is copying installed libraries (from node_modules) into the add-on. Nothing is compiled or bundled from authored source, so a source submission is not needed — the developer could ship the built XPI (with its package.json) and have it reviewed as a standard third-party-library (vendoring) submission instead. The build script can still be used locally.

The build is classified once, up front, by the build analysis (a single model request in the setup phase over the build-relevant files). This check reads that stored classification; when it is sca-redundant it raises one error, anchored at package.json, whose explanation is the analysis's reason.

When it fires The add-on is told a source-code submission is not required: nothing is compiled or bundled from authored source, so it should be submitted as a built XPI reviewed as a third-party-library review, declaring the bundled libraries (see the vendoring guide).

Source of truth: src/checks/rules/build-source-redundant.js (fed by src/build/analyze.js) and its entry in assets/registry.yaml.
