Experiment overrides a built-in API error

experiment-overrides-api

What it detects

An Experiment that declares an API which overrides a built-in Thunderbird API or grafts a new member onto a built-in namespace. Experiments may only add genuinely new APIs; they must not replace or extend existing ones.

For each API path declared under experiment_apis, the check resolves the path against the schema. A path that resolves to a registered experiment or to a top-level namespace absent from the real schema is a genuinely new API and is fine. Anything else collides with a built-in and is reported. The check is silent for non-Experiments.

When it fires The developer is told the named Experiment API overrides or extends a built-in Thunderbird API, that experiments may only add new APIs, and is asked to expose the functionality under a new, distinct API namespace.

Source of truth: src/checks/rules/experiment-overrides-api.js and its entry in assets/registry.yaml.
