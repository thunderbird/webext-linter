Build not derived from source error

build-not-from-source

What it detects

A source-code submission whose "build" does not actually build the add-on. Instead of compiling or bundling the reviewed source, the build only copies or zips files that are already present in the submission into the XPI. The shipped add-on is then assembled from phantom artifacts, not derived from the source the reviewer reads — either an oversight (the developer thinks zipping is the required build) or an attempt to have the reviewer vet a source the shipped XPI does not come from.

The build is classified once, up front, by the build analysis (a single model request in the setup phase over the build-relevant files). This check reads that stored classification; when it is not-from-source it raises one error, anchored at package.json, whose explanation is the analysis's reason.

When it fires The add-on is told the shipped XPI must be built from the reviewed source; a build that only copies or zips files already in the submission produces an add-on that is not derived from the source under review. The developer must provide a build that compiles or bundles the add-on from its source.

Source of truth: src/checks/rules/build-not-from-source.js (fed by src/build/analyze.js) and its entry in assets/registry.yaml.
