Library identified on a CDN info

find-lib-on-cdn

What it detects

This check reports a bundled JavaScript or CSS file that the known-library database (Mozilla dispensary's hashes.txt) did not recognise, but whose exact content hash is published on the jsDelivr CDN. It is a second-tier library identifier: jsDelivr indexes the raw SHA-256 of every file it serves, so a single content-addressed lookup (data.jsdelivr.com/v1/lookup/hash/<sha256>) names the exact package and version a bundle came from — no filename guessing, so it works even if the file was renamed.

A match means a known third-party library was shipped undeclared. The finding names the library and version and shows the canonical jsDelivr source URL, so the developer can add a VENDOR entry pairing the file with that source: URL and the bundled copy can be verified against the published release. Since a match is a precise, byte-verified release, the finding is advisory (info), not a blocker.

jsDelivr is uncurated, so a match must also clear a popularity bar to be trusted. A popular match joins the vendored family exactly like a Mozilla hash match: the file is treated as real third-party code (excluded from minified-code) and is audited for known vulnerabilities just like a declared dependency — an undeclared but vulnerable bundle is reported by vendor-vulnerable. A match that does not clear the bar is identified but untrusted: a minified one is rejected by untrusted-minified-library, a readable one is reviewed as the developer's own code and flagged (info) by untrusted-library. A bundle on neither the Mozilla DB nor the CDN is not treated as a library: if minified it falls to minified-code, if obfuscated to obfuscated-code, otherwise it is scanned as the developer's own source.

The lookup runs by default, caches every result on disk (hits and misses), and is best-effort — it is silently skipped when offline and makes no request for a file it has already resolved. Pass --cdn-lib-lookup false to disable it.

When it fires The add-on is told which third-party library and version each file was identified as on the CDN, and is given the jsDelivr source URL to declare in a VENDOR file — pairing the file with that source — so the bundled copy can be verified against the published release.
