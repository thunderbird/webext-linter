Unsupported build tool error

unsupported-build-tool

What it detects

A source-code submission whose build uses a package manager other than the two that are supported. Only npm and pnpm are accepted, because they share the same configuration format and keep the reviewable build surface small; other managers bring a much larger surface of runtime and plugin behaviour.

This check applies only to source-code submissions and works purely from the committed evidence, with no model or network. A disallowed manager is identified by its fingerprint at any depth in the build corpus — a lock file left behind by that tool, or the package.json field that names the manager explicitly. Because the tool policy is a single verdict, at most one finding is emitted. Pinning and the open-ended undeclared-source review are handled by separate checks.

When it fires The add-on is told it is built with an unsupported package manager and that only npm and pnpm are supported. It is asked to provide an npm or pnpm build — committing the matching lock file — so the dependency install can be reviewed against the declared dependencies.

Source of truth: src/checks/rules/unsupported-build-tool.js and its entry in assets/registry.yaml.
