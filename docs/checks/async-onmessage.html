Async listener in runtime.onMessage() warning

async-onmessage

What it detects

An async function passed straight to runtime.onMessage.addListener. An async listener always returns a Promise, which the messaging system reads as "I will respond" for every message — that claims the response channel and breaks other listeners.

The check looks only at the developer's own authored code; bundled, minified or otherwise non-authored files are skipped. It matches the specific call shape where a recognised API root's runtime.onMessage.addListener is invoked, and flags it only when the first argument is an async function or async arrow function. A plain (non-async) listener is left alone.

When it fires The add-on is told an asynchronous listener is attached to runtime.onMessage, that this is error-prone and must be avoided, with a link to the runtime messaging guide.
