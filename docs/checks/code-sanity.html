code-sanity

What it detects

This check runs a small set of high-signal correctness rules over the add-on's own JavaScript and reports likely mistakes such as a redeclared variable, a shadowed name, a duplicate object key or function argument, unreachable code, a self-assignment or self-comparison, or a malformed typeof check.

This check is opt-in: it runs only when --eslint is passed.

Only single-file correctness rules that rarely produce false positives are enabled. Stylistic and auto-fixable rules are deliberately left out because the tool is read-only and does not rewrite code, and undefined-symbol detection is off because WebExtension scripts share one global scope at runtime and would trip it constantly. Third-party, vendored, minified and obfuscated files are skipped — they are not the developer's authored source and are handled by the library and obfuscated-code checks instead.
