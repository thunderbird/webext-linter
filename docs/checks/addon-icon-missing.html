Add-on icon missing info

addon-icon-missing

What it detects

An add-on that declares no usable top-level icons entry ships no defined add-on icon, so Thunderbird shows the generic puzzle-piece placeholder in the Add-ons Manager. This is an advisory: defining an icon improves user acceptance.

An icon counts as defined only when the icons object holds at least one non-empty string path. An absent key, a non-object value, or an object with no string paths all mean "no icon defined". Static themes and dictionaries (language packs) are not represented by an add-on icon, so they are exempt.
