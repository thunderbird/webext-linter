Experiment with unrecognized API usage manual

experiment-unknown-api

What it detects

An Experiment add-on that calls a WebExtension API the schema does not recognize — a reachable call to an unknown namespace, an unknown member of a known namespace, or a member the schema marks unsupported. On an Experiment this usually means a typo in the add-on's own experiment schema rather than a genuine unsupported-API rejection.

The experiment schema's declared namespace is the sole authority for the API name, so if it does not match the API the code calls, that API is effectively undefined even when the manifest declares it. This check fires only when the submission is an Experiment and at least one such unrecognized usage exists, and escalates the whole add-on to manual review as a single reminder — the companion unknown-api finding already lists the specific APIs, so this one adds no locus and re-lists nothing.

When it fires The developer is told this Experiment calls WebExtension APIs that no schema defines, and — if these are their own Experiment APIs — is asked to verify that each experiment's schema declares the matching namespace, since the schema namespace is authoritative and a typo there leaves the API undefined even when the manifest declares it.
