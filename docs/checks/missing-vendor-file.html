Missing vendored file warning

missing-vendor-file

What it detects

A VENDOR entry that names a file which is not actually present in the submission — a stale entry or a file someone forgot to include.

Before the review, the VENDOR file is parsed and each declared path is checked against the packaged files; the absent ones are recorded. This is deliberately different from the package.json side, where a declared dependency whose file is missing is ignored on purpose — dependencies are installed or bundled at build time, so a not-yet-built submission may legitimately omit them. A VENDOR entry, by contrast, must point at a file that is really there. This check turns each missing VENDOR entry into a finding.

When it fires The add-on is told that one or more files listed in the VENDOR file are not present in the submission, and is asked to add each missing file or remove its entry from the VENDOR file.
