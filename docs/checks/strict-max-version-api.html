API newer than strict_max_version error

strict-max-version-api

What it detects

An add-on caps the Thunderbird versions it supports with strict_max_version, but calls a function or event that was only added in a later Thunderbird. No supported install would ever have that API.

Comparison is by major version: a strict_max_version like "140.*" covers every 140.x, so only a higher major counts as too new. When the manifest declares no strict_max_version there is no ceiling to violate, so the check does nothing.

Decision flow

flowchart TD

  B -- no --> S([skip])
  B -- yes --> C{"An API call in reachable code
resolving to a function/event?"}
  C -- no --> OK([no finding])
  C -- yes --> D["Look up the Thunderbird version
the API was added in"]
  D --> E{"Added in a version >
strict_max_version?"}
  E -- no --> OK
  E -- yes --> ERR([ERROR: API newer than
strict_max_version])

When it fires The add-on is told to raise strict_max_version or stop using the APIs added in a later Thunderbird.

Source of truth: src/checks/rules/strict-max-version-api.js and its entry in assets/registry.yaml.
