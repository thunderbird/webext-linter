Test the add-on and request testing information if needed manual

test-add-on

What it detects

This is a human-review item with no automated logic behind it. A reviewer exercises the add-on's functionality in a dedicated test profile to confirm it works as described.

While testing, the reviewer checks whether the review can proceed with what is already available, or whether extra information — such as login credentials for a website the add-on relies on — is needed to continue.

Decision flow

  class M manual;
When it fires The developer is asked for detailed instructions on how to test the add-on, and — if authentication is required — for a test username and password supplied through the reviewer-only Whiteboard field.

Source of truth: the manual-checks entry in assets/registry.yaml.
