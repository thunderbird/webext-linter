Possibly unused permissions manual

unused-permission-manual

What it detects

This check is a producer: it enumerates every declared named permission that a reachable API call does not provably require, and escalates each one for human review. A permission a reachable call provably needs is dropped. Host match patterns are out of scope here — they belong to the minimize-host-permissions check.

When the whole-add-on summary pass is enabled, the escalated permissions the registry has a recheck prompt for are handed to the unused-permission consumer to be re-judged with full context; every other unused permission (function-gated, where static analysis is authoritative) stays a manual-review reminder. Without the summary pass, all of them stand as the reminder. This one producer serves every add-on: the version-specific tabs wording (the D308076 fix in Thunderbird 154) is selected later, when the recheck rubric is assembled, from the add-on's strict_min_version.

When it fires The add-on is told that one or more declared permissions appear to be unused and is asked to remove any it does not need, or to explain which part of the add-on stops working without them.
