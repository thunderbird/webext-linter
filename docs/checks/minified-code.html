Source Archive (minified code) required error

minified-code

What it detects

This check flags the developer's own JavaScript when it ships minified — mechanically stripped of whitespace and meaningful names — and so cannot be reviewed as it stands. It detects minified line geometry: at least one very long line, dense on average.

A file that looks like a recognised third-party library is excluded here, because that is the missing-library check's concern, and a file that is obfuscated (deliberately hiding its behavior) is the obfuscated-code check's concern. This check fires only on machine-generated, merely-minified code that is not a known library.

When it fires The add-on is told the named files are minified or otherwise machine-generated and cannot be reviewed in that form, and is asked either to ship the original human-readable code or to upload a source archive with the build instructions needed to produce the final package.

The geometry heuristic cannot tell a build-tool bundle (webpack, tsc) from a minified library. For a built add-on like this, submit it as a source-code submission (--scs-root/--scs-source) so the readable sources are reviewed directly instead of the bundle. Obfuscated code stays rejected either way.
