Native messaging needs disclosure manual

native-messaging

What it detects

An add-on that declares the nativeMessaging permission, which lets it exchange messages with a native application running outside Thunderbird. That native host is not part of the package, so a reviewer must confirm the listing discloses what is exchanged, in keeping with the No Surprises policy. The trigger is deterministic, but the outcome is escalated to a human reviewer rather than reported as a finding.

The permission is the whole identity of this capability — the native messaging APIs do not work without it — so the check keys purely off whether the permission is declared, in either the permissions or the optional permissions list. No JavaScript scan is needed. If the manifest did not parse, or the permission is absent, nothing is escalated.

When it fires A reviewer is asked to confirm the add-on listing page discloses what information is exchanged with the local application and that it complies with the No Surprises policy. The native host itself does not need to be part of the package.

Source of truth: src/checks/rules/native-messaging.js and its entry in assets/registry.yaml.
