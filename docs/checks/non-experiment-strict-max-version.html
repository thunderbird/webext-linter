Unnecessary strict_max_version warning

non-experiment-strict-max-version

What it detects

A regular MailExtension that pins a maximum Thunderbird version it will install on. The WebExtension APIs are stable and designed to keep add-ons forward-compatible, so a ceiling only blocks installs on newer Thunderbird with no benefit.

The check looks at the manifest only. Experiments are left alone because they reach into Thunderbird internals and legitimately need a maximum version, and a non-Experiment that omits the ceiling is fine. It fires only when a non-Experiment manifest actually declares one.

When it fires The add-on is told it pins a maximum version that is usually unnecessary, since the WebExtension APIs stay forward-compatible across Thunderbird releases, and that the ceiling only blocks installs on newer Thunderbird.

Source of truth: src/checks/rules/non-experiment-strict-max-version.js and its entry in assets/registry.yaml.
