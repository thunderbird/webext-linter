Synchronous XMLHttpRequests error

sync-xhr

What it detects

A synchronous XMLHttpRequest — an open call whose third argument is the boolean false. A synchronous request blocks the UI thread until it completes, which can lock up the interface.

The check looks only at the developer's own authored code; bundled, minified or otherwise non-authored files are skipped, since a library's own request is not the developer's concern. Within authored code it examines open member calls that carry an explicit boolean async flag, and reports the ones where that flag is false. An asynchronous call, or one without an explicit boolean flag, is left alone.

When it fires The add-on is told it makes a synchronous XMLHttpRequest that can lock up the user interface, and asked to use an asynchronous request instead.
