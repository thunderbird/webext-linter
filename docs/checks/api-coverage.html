Static analysis coverage info

api-coverage

What it detects

Places where the static analysis could not fully follow the code because an API was reached dynamically or through an alias that cannot be resolved without running the add-on. These are coverage gaps in the runner's own analysis, not problems with the add-on itself.

It reads the per-file usage records gathered during parsing and forwards each recorded limitation, using the parser's own diagnostic as the message. Only files in the pure WebExtension tree are considered; Experiment and core code as well as unreachable dead code are excluded. A file that failed to parse outright is handled by the separate unparsable-file check rather than here.

When it fires The report notes that static-analysis coverage was limited at the flagged location, quoting the specific reason the access could not be resolved, so a reviewer knows that part of the code was not fully checked automatically.

Source of truth: src/checks/rules/api-coverage.js and its entry in assets/registry.yaml.
