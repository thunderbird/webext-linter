Missing manifest error

manifest-missing

What it detects

There is no manifest.json at the add-on root at all, so the submission is not a valid add-on. This fires only when the manifest is genuinely absent.

A manifest that is present but unparsable is a different situation, reported by the invalid-manifest-JSON check — so this check stays silent in that case to avoid a duplicate finding.

Decision flow

flowchart TD

  B -- "absent entirely" --> ERR([ERROR: missing manifest])

When it fires The add-on is told there is no manifest.json at the add-on root, with a link to the WebExtension API documentation.
