Background page script needs type module error

background-page-module

What it detects

A background page is an HTML file that loads its scripts through <script src> tags. When such a tag points at a JavaScript file that uses ES module syntax but the tag is not declared as a module, Thunderbird loads the file as a classic script and the module syntax fails, breaking the background page.

The check reads the background page declared in the manifest, walks its external script tags, and follows each one that resolves to a packaged file. The sibling background-module check covers the manifest's background scripts and service worker. Inline script blocks and tags that already declare a module type, or whose target is remote or unresolved, are out of scope.

When it fires The add-on is told the flagged background page loads a script that uses ES module syntax while the script tag is not declared type="module", so the page will fail to load, and must add type="module" to the tag.

Source of truth: src/checks/rules/background-page-module.js and its entry in assets/registry.yaml.
