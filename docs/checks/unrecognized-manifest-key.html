Unrecognized manifest key info

unrecognized-manifest-key

What it detects

A top-level key in manifest.json that the schema does not define. Thunderbird simply ignores such keys, so this is reported only as information, not as a defect that breaks loading.

It runs only when the manifest is present and parsed — a missing or unparsable manifest is left to the manifest error checks. It also stays silent if the schema lists no valid keys at all. Otherwise it walks every top-level key and reports each one the schema does not know, pointing at the relevant line in manifest.json where possible. A key that names one of the add-on's own experiment_apis entries is excepted — it is configuration the add-on's Experiment reads, so the developer owns it.

When it fires The add-on is told which top-level manifest keys are unknown and will be ignored by Thunderbird, with a link to the WebExtension API documentation.

Source of truth: src/checks/rules/unrecognized-manifest-key.js and its entry in assets/registry.yaml.
