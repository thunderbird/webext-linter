Unknown or unsupported WebExtension API error

unknown-api

What it detects

Calls to WebExtension APIs (browser.*, messenger.*, chrome.*) that Thunderbird's annotated schema does not provide: an unknown namespace, an unknown member of a known namespace, or a member that exists but is explicitly marked unsupported.

Only the plain WebExtension code is checked — Experiment and privileged code legitimately use Thunderbird internals rather than the WebExtension schema, and dead code never runs, so both are excluded.

Decision flow

flowchart TD
  A["An API call in the source"] --> B{"In reachable WebExtension code?"}
  B -- no --> S([skip])
  B -- yes --> C["Resolve the call against
Thunderbird's schema"]
  C --> D{"Namespace unknown?"}
  D -- yes --> ERR([ERROR: unknown or
unsupported API])
  D -- no --> E{"Member unknown?"}
  E -- yes --> ERR
  E -- no --> F{"Marked 'unsupported'
in the schema?"}
  F -- yes --> ERR
  F -- no --> OK([no finding])

When it fires The add-on is told the named API is not supported in the targeted Thunderbird version, with a link to the WebExtension API documentation.

Source of truth: src/checks/rules/unknown-api.js and its entry in assets/registry.yaml.
