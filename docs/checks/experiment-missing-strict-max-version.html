Experiment missing strict_max_version error

experiment-missing-strict-max-version

What it detects

A valid Experiment that does not pin a strict_max_version. Experiments call internal Thunderbird APIs that change between major versions, so without a maximum supported version they can break silently when the user upgrades.

This is a deterministic-phase check, so it runs only once an experiment has been accepted onto the normal review path — either because experiments are allowed or because the bundled copy is a pristine upstream draft. An unsupported experiment never reaches here, because it short-circuits to the experiment-not-allowed check. The check is silent for non-Experiments and for Experiments that already declare a maximum.

When it fires The developer is told the add-on uses Experiment APIs but declares no strict_max_version, and that because experiments rely on internal APIs that change between major Thunderbird versions they must set the gecko strict_max_version in browser_specific_settings.

Source of truth: src/checks/rules/experiment-missing-strict-max-version.js and its entry in assets/registry.yaml.
