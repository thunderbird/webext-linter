Obfuscated code is not permitted error

obfuscated-code

What it detects

This check flags the developer's own JavaScript when it ships obfuscated — code deliberately written to hide its behavior — and so cannot be reviewed as it stands. It detects javascript-obfuscator-style hex identifiers in bulk and eval-or-Function-of-decoded-string packers.

A file that looks like a recognised third-party library is excluded here, because that is the missing-library check's concern, and a file that is merely minified (not obfuscated) is the minified-code check's concern. The detection has high precision but partial recall, so it cannot catch every obfuscator.

When it fires The add-on is told the named files are obfuscated and that obfuscated code is not permitted — whether or not the add-on is hosted on addons.thunderbird.net — as it violates the Add-on Policy and is subject to being blocked. It is asked to ship the original, human-readable code directly.

Source of truth: src/checks/rules/obfuscated-code.js and its entry in assets/registry.yaml.
