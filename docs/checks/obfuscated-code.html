Source Archive (minified or obfuscated code) required error

obfuscated-code

What it detects

This check flags the developer's own JavaScript when it ships minified or obfuscated and so cannot be reviewed as it stands. It detects minified line geometry — at least one very long, dense line — javascript-obfuscator-style hex identifiers in bulk, and eval-or-Function-of-decoded-string packers.

A file that looks like a recognised third-party library is excluded here, because that is the missing-library check's concern; this check only fires on machine-generated code that is not a known library. The detection has high precision but partial recall, so it cannot catch every obfuscator.

When it fires The add-on is told the named files are minified, obfuscated or otherwise machine-generated and cannot be reviewed in that form, and is asked either to ship the original human-readable code or to upload a source archive with the build instructions needed to produce the final package.

Source of truth: src/checks/rules/obfuscated-code.js and its entry in assets/registry.yaml.
