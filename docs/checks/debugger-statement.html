Debugger statement error

debugger-statement

What it detects

An unconditional debugger statement left in shipped code. Such a statement halts execution whenever developer tools are open, so it does not belong in a released add-on.

The check looks only at the developer's own authored code; bundled, minified or otherwise non-authored files are skipped. A debugger statement placed inside an if is treated as gated behind a config flag and allowed — since a debugger statement cannot be made conditional any other way, an enclosing if is the signal of intent. Only statements that always execute are flagged.

When it fires The add-on is told it contains an unconditional debugger statement and is asked to remove it or move it behind a config flag.
