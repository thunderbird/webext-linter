Disguised data transmission (navigation) error

disguised-navigation

What it detects

Data smuggled out of the add-on by hiding it inside a page navigation. Navigating the location to a remote address — through location.assign or location.replace — built up with runtime data appended to it sends that data to the remote host while looking like an ordinary page navigation.

A navigation URL is not a channel meant for transmitting data. Disguising a data send this way is treated as an error regardless of any user consent. The check looks only at the developer's own JavaScript and fires on the strong case: a covert navigation target to a remote host with a user-data API call (message content, contacts, credentials and the like) sitting in it. The weaker case — merely a runtime value appended to a remote URL, common in legitimate code — is left to the disguised-transmission LLM check. Navigating to a plain static remote URL with no data attached is not a finding.

When it fires The add-on is told it appears to send data through a page navigation — a channel not meant for transmission — and that disguising a send this way is not allowed even with consent. It is asked to use a normal request and disclose what is sent.
