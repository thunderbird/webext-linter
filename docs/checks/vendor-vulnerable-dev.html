Vulnerable build dependency auto

vendor-vulnerable-dev

What it detects

A pinned npm build dependency whose version has published security advisories. Build dependencies are never shipped with the add-on, but this check runs only for a source-code submission, where the reviewer builds the add-on from source — so a vulnerable build tool runs on the reviewer's own machine.

Before the review, every pinned npm development dependency is audited once against the OSV vulnerability database over the network. Each package with known advisories is recorded with the advisory ids, the affected version, the fixed versions, and a severity band. This check reads that result and emits one finding per vulnerable build dependency, anchored at its declaration line. Its severity is auto: rather than a fixed level, each finding takes its severity from the advisory band — high or critical becomes an error, moderate or medium becomes a warning, and anything else (low or unknown) is informational.

When it fires The add-on is told a build dependency has known security vulnerabilities, with the affected version, the advisory ids, and the severity, and that although it is not shipped, reviewers build the add-on from source so it runs on the reviewer's machine during the build. It is asked to update the dependency to a fixed release and resubmit.

Source of truth: src/checks/rules/vendor-vulnerable-dev.js and its entry in assets/registry.yaml.
