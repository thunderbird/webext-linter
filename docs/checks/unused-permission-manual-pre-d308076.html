Possibly unused permissions (pre-D308076) manual

unused-permission-manual-pre-d308076

What it detects

This check is a producer: it enumerates every declared named permission that a reachable API call does not provably require, and escalates each one for human review. A permission a reachable call provably needs, or one that gates no callable API at all and so is justified by its mere presence, is dropped. Host match patterns are out of scope here — they belong to the minimize-host-permissions check.

It is the pre-D308076 sibling: it runs for add-ons whose minimum Thunderbird version is below 154, or absent or unparsable, which are treated as possibly running on older versions by default. The other sibling producer covers versions at or above 154; exactly one of the two fires per add-on. It feeds the more relaxed unused-permission-pre-d308076 recheck consumer, whose rubric keeps the tabs permission justified whenever the code filters a tab query by url or title anywhere — because before the fix that needs tabs even on the add-on's own pages. When the whole-add-on summary pass is enabled, the escalated permissions are handed to that consumer; otherwise they stand as a single manual-review reminder.

When it fires The add-on is told that one or more declared permissions appear to be unused and is asked to remove any it does not need, or to explain which part of the add-on stops working without them.
