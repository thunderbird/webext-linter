Missing permission error

missing-permission

What it detects

An add-on requires a named permission but does not declare it in the manifest. A permission is required either because a reachable API call needs it, or because the add-on declares a script-injection manifest key that implies it — a declared compose_scripts key requires compose, and message_display_scripts requires messagesModify. The shared permission analysis matches each reachable API call, and each such declared key, to the permission the schema says it needs, and flags the ones the manifest does not request.

The check only ever proves a permission is required. It never reports a declared permission as unused — a permission can be needed only to read a permission-gated property of returned data, which a static scan cannot confirm. (That advisory judgement is left to the optional --llm-review LLM pass.) An API that needs a manifest key rather than a permission is handled by the missing-manifest-key check.
