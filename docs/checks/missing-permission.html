Missing permission error

missing-permission

What it detects

An add-on calls an API that requires a named permission, but that permission is not declared in the manifest. The shared permission analysis matches each reachable API call to the permission the schema says it needs, and flags the ones the manifest does not request.

The check only ever proves a permission is required. It never reports a declared permission as unused — a permission can be needed only to read a permission-gated property of returned data, which a static scan cannot confirm. (That advisory judgement is left to the optional --llm-review LLM pass.) An API that needs a manifest key rather than a permission is handled by the missing-manifest-key check.

Decision flow

flowchart TD
  A["An API call in reachable code"] --> B{"Does the schema say it requires
a named permission?"}
  B -- no --> OK([no finding])
  B -- yes --> C{"Is that permission declared
in the manifest?"}
  C -- yes --> OK
  C -- no --> ERR([ERROR: missing permission])
