Unused, hidden or duplicate files or folders (recheck) error

unused-files-recheck

What it detects

This is the post-summary recheck consumer for the unused-files check. It re-judges the files that the per-site analysis could not resolve to a loader — the residual ambiguous items — now with the whole add-on in view rather than one site at a time.

It runs only in the post-summary phase. When a full add-on summary is produced, those unresolved files are appended to it under this entry's summary rubric and re-judged: the model decides whether anything anywhere in the add-on loads each file (a static or dynamic import, a getURL or script/CSS injection, an HTML or CSS reference, a consumed data entry, or a plausible runtime-built path). With no summary, nothing is handed over, this check emits nothing, and unused-files' own manual items stand.

When it fires The add-on is told the listed files or folders appear unused or hidden, that they complicate review and bloat the package, and is asked to fix its packaging so they are no longer included.
