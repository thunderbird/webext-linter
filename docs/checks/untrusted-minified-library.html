Source Archive (unverified library) required error

untrusted-minified-library

What it detects

A bundled file whose bytes match a pinned upstream release — recognised via the Mozilla hash database, the jsDelivr CDN, or a declared VENDOR source — but whose package could not be confirmed to be a widely-used library, and which is also minified or otherwise machine-generated.

Such a file is both untrusted, so it does not earn the trusted-library review exemption, and unreviewable in this form, because the shipped bytes are machine-generated rather than readable. It cannot be reviewed as authored code the way a readable untrusted library can, so the developer must ship the original, human-readable code or upload a source code archive with build instructions.

Decision flow

When it fires The add-on is told each named file matches a known upstream release but could not be confirmed to be a widely-used library, so it is not trusted as a reviewed dependency, and that because it is minified or otherwise machine-generated it cannot be reviewed in this form. It is asked to ship the original, human-readable code directly, or to upload the original sources as a source-code submission — with the build instructions needed to reproduce the shipped package — during submission.

Source of truth: src/checks/rules/untrusted-minified-library.js, src/checks/lib/bundled.js, and its entry in assets/registry.yaml.
